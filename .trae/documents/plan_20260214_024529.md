## 问题分析
当前设计只考虑了fund_nav表，但数据库有多个表，每个表的最新日期不同。

## 解决方案

### 1. 修改日期选择对话框，支持多表同步

```
┌─────────────────────────────────────────────┐
│ 数据同步                                     │
├─────────────────────────────────────────────┤
│ 请选择要同步的数据表：                        │
│                                             │
│ ☑ 基金基本信息 (fund_info)                  │
│   └ 全量同步，获取所有基金                    │
│                                             │
│ ☑ 基金净值 (fund_nav)                       │
│   └ 增量同步：2025-11-22 至 2026-02-14      │
│   └ 最新日期：2025-11-21                    │
│                                             │
│ ☐ 基金持仓 (fund_holding)                   │
│   └ 增量同步：2025-10-01 至 2026-02-14      │
│   └ 最新日期：2025-09-30                    │
│                                             │
│ ☐ 基金经理 (fund_manager)                   │
│   └ 全量同步                                │
│                                             │
│ 同步方式：○自动增量  ○指定日期范围            │
│                                             │
│ [取消] [开始同步]                            │
└─────────────────────────────────────────────┘
```

### 2. 每个表单独显示
- 表名和描述
- 同步类型（全量/增量）
- 最新日期
- 预计同步范围

### 3. 用户可选择
- 勾选要同步的表
- 选择同步方式（自动增量/指定日期）

### 4. 后端支持
- 获取所有表的最新日期
- 支持按表同步

## 实施步骤
1. 添加获取所有表状态的API
2. 修改前端对话框，显示多表选择
3. 修改同步逻辑，支持选择性同步
